## 🎯 Sobre o Projeto

**Nome:** FuseHotel
**Tipo:** Sistema de Gestão Hoteleira Completo
**Arquitetura:** Monorepo com npm workspaces
**Status:** ✅ Produção (consolidado e funcional)

### Propósito
Sistema completo para gerenciamento de hotel incluindo:
- Frontend web responsivo para clientes e administradores
- Backend API RESTful robusto
- Sistema de reservas online com cálculo automático
- Painel administrativo completo
- Autenticação JWT com refresh tokens
- Upload de imagens, newsletter, contato

## 🔐 Autenticação e Segurança

### JWT Strategy
- **Access Token:** 15 minutos (HS256)
- **Refresh Token:** 7 dias (armazenado no DB)
- **Middleware:** `auth.middleware.ts` valida tokens
- **Role Middleware:** `role.middleware.ts` verifica permissões

### Roles e Permissões
- **ADMIN:** Acesso total (CRUD de tudo)
- **MANAGER:** Gerencia acomodações, reservas, promoções
- **CUSTOMER:** Visualiza e cria apenas suas próprias reservas

### Segurança
- Helmet.js (security headers)
- CORS configurado
- Rate limiting (100 req/15min geral, 5 req/min auth)
- Bcrypt (10 rounds) para senhas
- Validação Zod em todas as entradas
- SQL Injection protection (Prisma)
- XSS protection
- Upload validado (tipo, tamanho max 5MB)

## 🌐 API Endpoints

### Autenticação (/api/auth)
- `POST /api/auth/register` - Registrar novo usuário
- `POST /api/auth/login` - Login (retorna access + refresh token)
- `POST /api/auth/refresh` - Renovar access token
- `POST /api/auth/logout` - Logout (invalida refresh token)
- `POST /api/auth/forgot-password` - Solicitar reset de senha
- `POST /api/auth/reset-password` - Resetar senha com token
- `PUT /api/auth/change-password` - Alterar senha (autenticado)

### Usuários (/api/users)
- `GET /api/users` - Listar usuários (ADMIN)
- `GET /api/users/profile` - Perfil do usuário logado
- `PUT /api/users/profile` - Atualizar perfil
- `GET /api/users/:id` - Obter usuário por ID (autenticado)
- `DELETE /api/users/:id` - Deletar usuário (ADMIN)

### Acomodações (/api/accommodations)
- `GET /api/accommodations` - Listar (público, com filtros)
- `GET /api/accommodations/:id` - Obter por ID
- `GET /api/accommodations/slug/:slug` - Obter por slug
- `POST /api/accommodations` - Criar (ADMIN/MANAGER)
- `PUT /api/accommodations/:id` - Atualizar (ADMIN/MANAGER)
- `DELETE /api/accommodations/:id` - Deletar (ADMIN)

### Reservas (/api/reservations)
- `GET /api/reservations` - Listar todas (ADMIN/MANAGER)
- `GET /api/reservations/my-reservations` - Minhas reservas (autenticado)
- `GET /api/reservations/:id` - Obter por ID (autenticado)
- `GET /api/reservations/code/:code` - Obter por código
- `POST /api/reservations` - Criar reserva
- `POST /api/reservations/:id/cancel` - Cancelar (autenticado)

### Promoções (/api/promotions)
- `GET /api/promotions` - Listar (público)
- `GET /api/promotions/:id` - Obter por ID
- `GET /api/promotions/slug/:slug` - Obter por slug
- `POST /api/promotions` - Criar (ADMIN/MANAGER)

### Outros
- `/api/settings` - Configurações do sistema
- `/api/newsletter` - Newsletter
- `/api/contact` - Mensagens de contato
- `/api/upload` - Upload de imagens
- `/api/health` - Health check

## 📝 Padrões de Código

### Estrutura de Resposta API
```typescript
{
  success: boolean,
  data?: T,
  message?: string,
  errors?: ValidationError[],
  meta?: {
    timestamp: string
  }
}
```

### Convenções de Nomenclatura
- **Arquivos:** kebab-case (`user.controller.ts`, `auth.middleware.ts`)
- **Componentes React:** PascalCase (`UserProfile.tsx`, `LoginForm.tsx`)
- **Funções/variáveis:** camelCase (`getUserById`, `isAuthenticated`)
- **Constantes:** SCREAMING_SNAKE_CASE (`JWT_SECRET`, `MAX_FILE_SIZE`)
- **Tipos/Interfaces:** PascalCase (`User`, `CreateUserDto`, `ApiResponse<T>`)

### Organização de Imports
```typescript
// 1. Node modules
import express from 'express';

// 2. Packages locais (@fusehotel/shared)
import { User, UserRole } from '@fusehotel/shared';

// 3. Módulos internos
import { authMiddleware } from '../middlewares/auth.middleware';
```

## 📚 Contexto para Desenvolvimento

### Ao trabalhar com Backend:
1. **Sempre use** Prisma Client para queries (nunca SQL raw)
2. **Sempre valide** inputs com Zod schemas do @fusehotel/shared
3. **Sempre use** middlewares de auth/role quando aplicável
4. **Sempre retorne** ApiResponse padronizado
5. **Sempre logue** erros com Winston logger

### Ao trabalhar com Frontend:
1. **Sempre use** componentes ShadcnUI quando disponível
2. **Sempre use** TanStack Query para fetching
3. **Sempre use** React Hook Form + Zod para formulários
4. **Sempre use** tipos do @fusehotel/shared
5. **Sempre use** Tailwind para estilização (não CSS inline)

### Ao criar novos recursos:
1. **Backend:** Controller → Service → Route → Middleware (se necessário)
2. **Frontend:** Tipo → Hook → Componente → Página
3. **Database:** Migration → Seed (se necessário) → Update types
4. **Shared:** Adicionar tipos/validadores se compartilhados

### "Prisma Client not generated"
```bash
cd apps/api
npx prisma generate
```

### "Port 3000/3001 already in use"
```bash
# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# Linux/Mac
lsof -ti:3000 | xargs kill -9
```

### "Cannot find module '@fusehotel/shared'"
```bash
# Root do monorepo
npm install
```

### JWT token expirado
- Access token expira em 15min
- Frontend deve usar `/auth/refresh` automaticamente
- Axios interceptors já configurados
