## 📋 Visão Geral do Projeto

**FuseHotel** é um sistema completo de gerenciamento hoteleiro construído como monorepo TypeScript full-stack. O projeto está em produção em [perolahotel.com](https://perolahotel.com) e inclui:

- Backend REST API com Express, Prisma e PostgreSQL
- Frontend React SPA com Vite e Tailwind CSS
- Sistema de customização visual completo (20+ customizadores)
- Infraestrutura Docker com CI/CD via GitHub Actions
- Deploy automatizado em VPS (72.60.10.112)

## 🔐 Autenticação e Autorização

### JWT Flow

```
1. Login → Access Token (15min) + Refresh Token (7d, httpOnly)
2. Requests → Bearer Token no header
3. Token expirado → POST /auth/refresh com cookie
4. Logout → Revoga refresh token
```

### Roles

- **ADMIN**: Acesso total (CRUD de tudo)
- **MANAGER**: Gestão de reservas e acomodações
- **CUSTOMER**: Visualizar e criar reservas próprias

### Middleware de Auth

```typescript
// apps/api/src/middlewares/auth.middleware.ts

// Verificar se está autenticado
authenticate(req, res, next)

// Verificar role específica
authorize(...roles: Role[])

// Exemplo de uso em rota:
router.post('/', authenticate, authorize('ADMIN'), createAccommodation)
```

### Protected Routes (Frontend)

```typescript
// apps/web/src/App.tsx
<Route element={<ProtectedRoute allowedRoles={['ADMIN']} />}>
  <Route path="/admin/*" element={<AdminLayout />} />
</Route>
```

## 🎯 Padrões de Código

### Backend

**Controller Pattern**:

```typescript
// apps/api/src/controllers/example.controller.ts
export const getItems = async (req: Request, res: Response) => {
  try {
    const items = await itemService.findAll(req.query)
    res.json(items)
  } catch (error) {
    throw new AppError('Erro ao buscar items', 500)
  }
}
```

**Service Pattern**:

```typescript
// apps/api/src/services/example.service.ts
export const itemService = {
  async findAll(filters: any) {
    return await prisma.item.findMany({
      where: filters,
      include: { relations: true }
    })
  },

  async findById(id: string) {
    const item = await prisma.item.findUnique({ where: { id } })
    if (!item) throw new NotFoundError('Item não encontrado')
    return item
  }
}
```

**Route Pattern**:

```typescript
// apps/api/src/routes/example.routes.ts
import { Router } from 'express'
import { authenticate, authorize } from '../middlewares/auth.middleware'
import * as controller from '../controllers/example.controller'

const router = Router()

router.get('/', controller.getItems)
router.get('/:id', controller.getItem)
router.post('/', authenticate, authorize('ADMIN'), controller.createItem)
router.put('/:id', authenticate, authorize('ADMIN'), controller.updateItem)
router.delete('/:id', authenticate, authorize('ADMIN'), controller.deleteItem)

export default router
```

```typescript
// apps/web/src/hooks/useExample.ts
export function useExample() {
  const queryClient = useQueryClient()

  const { data, isLoading } = useQuery({
    queryKey: ['examples'],
    queryFn: () => api.get('/examples').then(res => res.data)
  })

  const createMutation = useMutation({
    mutationFn: (newExample) => api.post('/examples', newExample),
    onSuccess: () => {
      queryClient.invalidateQueries(['examples'])
      toast.success('Criado!')
    }
  })

  return { data, isLoading, createMutation }
}
```

## 🔍 Troubleshooting Guide

### Problema: Erro de conexão com banco

**Sintomas**: `PrismaClientInitializationError`

**Solução**:
```bash
# Verifique se PostgreSQL está rodando
docker-compose -f docker-compose.dev.yml ps

# Verifique DATABASE_URL no .env
# Formato: postgresql://user:pass@host:port/dbname

# Teste conexão
npm run prisma:studio
```

### Problema: Migrações falhando

**Sintomas**: Erro ao rodar `prisma:migrate`

**Solução**:
```bash
# Opção 1: Reset (PERDE DADOS!)
npm run prisma:migrate -- reset

# Opção 2: Criar nova migração de fix
npm run prisma:migrate -- --name fix_schema

# Opção 3: Push direto (dev only)
npx prisma db push
```

### Problema: CORS error no frontend

**Sintomas**: `Access-Control-Allow-Origin` error

**Solução**:
```typescript
// apps/api/src/server.ts
// Verifique se CORS está configurado corretamente
app.use(cors({
  origin: process.env.FRONTEND_URL || 'http://localhost:3000',
  credentials: true
}))
```

### Problema: JWT não está sendo enviado

**Sintomas**: 401 Unauthorized apesar de estar logado

**Solução**:
```typescript
// apps/web/src/lib/api.ts
// Verifique se axios está configurado para enviar credentials
axios.defaults.withCredentials = true

// E se o interceptor está adicionando o token
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

### Problema: Upload de imagem falhando

**Sintomas**: 400 ou 413 error

**Solução**:
```bash
# Verifique MAX_FILE_SIZE no .env da API
# Verifique formato do arquivo (jpg, jpeg, png, gif, webp)
# Verifique se diretório uploads existe e tem permissão

mkdir -p apps/api/uploads
chmod 755 apps/api/uploads
```

### Problema: Docker build falhando

**Sintomas**: Erro durante `docker-compose up`

**Solução**:
```bash
# Limpe cache do Docker
docker-compose down -v
docker system prune -a

# Rebuilde sem cache
docker-compose -f docker-compose.dev.yml build --no-cache
docker-compose -f docker-compose.dev.yml up -d
```

## 🎓 Convenções e Boas Práticas

### Commits

Seguir [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: adiciona nova funcionalidade
fix: corrige bug
docs: atualiza documentação
style: formatação (sem mudança de lógica)
refactor: refatoração
test: adiciona ou corrige testes
chore: mudanças em build, configs
```

### Branches

```
main              # Produção (protegida)
develop           # Desenvolvimento
feature/nome      # Nova feature
fix/nome          # Correção de bug
hotfix/nome       # Correção urgente em produção
```

### TypeScript

- Sempre tipar parâmetros e retornos
- Usar interfaces para objetos complexos
- Evitar `any` (usar `unknown` se necessário)
- Usar type guards quando apropriado

### React

- Componentes funcionais (hooks)
- Props sempre tipadas
- Use memo apenas quando necessário (performance)
- Custom hooks para lógica reutilizável
- Context API para estado global (auth, theme)

### Prisma

- Sempre usar includes/select para otimizar queries
- Usar transactions para operações múltiplas
- Validar dados antes de inserir
- Use soft deletes onde apropriado
